## 修复 pages/index/index.vue 智能问答机器人回复后不滚动问题

### 根因
1. **第1521行中文乱码**：`this.在 = 'msg-' + ...`（应为 `this.scrollToMsg`），导致 `scrollToMsg` 从未被正确赋值，且触发 `ReferenceError: scrollToMsg is not defined`。
2. **scroll-into-view 相同值不触发滚动**：流式回复中 `idx` 固定，每次 delta 设同一个 `scrollToMsg` 值，微信小程序只在值变化时才滚动。
3. **DOM 更新与滚动同周期**：内容变长和滚动设置在同一个 Vue 渲染周期，DOM 未更新完就滚动，位置不准。
4. **CSS overflow-y: auto 与 scroll-view 冲突**：scroll-view 是原生组件自管滚动，CSS overflow 可能干扰。

### 修复方案（8处改动）
1. 模板：`:scroll-into-view="scrollToMsg"` → `:scroll-top="scrollTopVal"` + `:scroll-with-animation="false"`（流式高频动画会卡）
2. data：新增 `scrollTopVal: 0`
3. 修复第1521行 `this.在` → `this.scrollToMsg`
4. 流式回复（第1653行）：`this.scrollTopVal = this.scrollTopVal + 1`（递增确保每次值变化）
5. 其他4处滚动点（发送/测试/卡片/降级回复）：`this.$nextTick(() => { this.scrollTopVal = this.scrollTopVal === 99999 ? 99998 : 99999 })`
6. CSS：删除 `.chat-messages` 的 `overflow-y: auto`

### 经验总结
- 微信小程序 scroll-view 的 `scroll-into-view` 相同值不触发滚动，流式场景必须用 `scroll-top` + 数值递增。
- `scroll-top` 设大数（如99999）可强制滚到底部，但相同值不触发，需用递增或值切换技巧。
- 流式回复场景应关闭 `scroll-with-animation`，delta 频率高时动画会卡顿。

## 修复 503 错误未被重试的 bug（16:56）
**根因**：`wx.cloud.extend.AI.bot.sendMessage()` 调用在 `try` 块**外部**，当 sendMessage 本身抛出 503 时，直接跳到外层 catch 显示降级回答，重试逻辑（maxRetries=5，指数退避）根本没执行。
**修复**：将 sendMessage 调用移入 try 块内，使 503 能被 `catch(streamErr)` 捕获并设置 `needRetry=true`，触发重试。
**经验**：异步 SDK 调用必须在 try 块内才能被对应的 catch 捕获重试，放在 try 外部的 await 抛出会直接冒泡到最外层 catch。
