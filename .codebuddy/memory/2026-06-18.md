## pages/index/index.vue 修复 onQuickQuestion 报错

- 报错：`ReferenceError: q is not defined` at `onQuickQuestion (index.vue:1501)`
- 根因：方法签名 `onQuickQuestion(e)` 参数名为 `e`，但方法体用了 `q`；且 uni-app 编译器对模板直接传字符串参数（如 `@tap="onQuickQuestion('卡片')"`）处理不可靠。
- 先简单修复：参数名 `e` → `q`。
- 应用户要求进一步改为 dataset 传值方式（与 `onChatLinkTap` 一致，更可靠）：
  - 模板（第604-607行）：`@tap="onQuickQuestion('卡片')"` → `data-q="卡片" @tap="onQuickQuestion"`
  - 方法（第1499行）：`onQuickQuestion(e) { const q = e.currentTarget.dataset.q; ... }`
- 无 lint error，仅预先存在的未使用变量 HINT。
